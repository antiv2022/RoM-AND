Some vanilla AND tags have been removed or replaced, See "Deprecated XML Tags.txt" for more information.

Listed here is all the new features for XML that should be used when merging with vanilla AND. Examples of tag structure are included.

Civ4UnitInfos.xml:-------------------------------------------------------------------------------------------------------------------------

<iVolleyEfficiency>100</iVolleyEfficiency> /////Percent of unit strength used in power calculation. Further influenced by VOLLEY_STRENGTH in Inthegrave_GlobalDefines.xml
<iVolleyRange>5</iVolleyRange> /////Base number of tiles for the ranged attack. If set to 0 the ranged attack is turned off.
<iVolleyAccuracy>85</iVolleyAccuracy> /////Base percent chance for the ranged attack to hit on the first tile.
<iVolleyAccuracyMin>15</iVolleyAccuracyMin> /////Accuracy for the last tile. Accuracy scales between this and the above value.
<iVolleyRounds>1</iVolleyRounds> /////How many shots fired using the same above stats.


Civ4BuildingInfos.xml:--------------------------------------------------------------------------------------------------------------------------

<iCrime>-1</iCrime> ///// Increases or decreases crime by this value.
<iCrimePerPop>0</iCrimePerPop> ///// Gives crime points per population milestone. 100 means 1 crime per population and 50 gives 1 crime every 2 population


List of free buildings the building will give to all cities upon being built.
<ExtraFreeBuildings>
	<ExtraFreeBuilding>
		<BuildingClassType>BUILDINGCLASS_GRAVEYARD</BuildingClassType>
		<bConnected>1</bConnected> ///// Whether or not the free building is destroyed when it's master is destroyed.
		<bContinuous>0</bContinuous> ///// Whether or not a free building is automatically reconstructed when missing. If 0 then the building is only built once by the master when completed.
	</ExtraFreeBuilding>
	<ExtraFreeBuilding>
		<BuildingClassType>BUILDINGCLASS_IRRIGATION_CANALS</BuildingClassType>
		<bConnected>0</bConnected>
		<bContinuous>1</bContinuous>
	</ExtraFreeBuilding>
</ExtraFreeBuildings>


CIV4SpawnInfos.xml:--------------------------------------------------------------------------------------------------------------------------------------------------------------

iMinAreaPlotsPerPlayerUnit /////Minimum number of area plots per unit owned by spawning player within the area before spawn is disabled.
iMinAreaPlotsPerUnitType /////Minimum number of area plots per "spawn unit type" within the area before spawn is disabled.

Leader bigotry:

Race and gender of the current leader:
<Gender>GENDER_MALE</Gender>
<Race>RACE_WHITE</Race>

Genders and races the leader can give negative attitudes for:
<SexistAttitudes>
	<SexistAttitude>
		<Gender>GENDER_FEMALE</Gender>
		<iAttitudeChange>-2</iAttitudeChange>
	</SexistAttitude>
</SexistAttitudes>

		<iAttitudeChange>-1</iAttitudeChange>
	</RacistAttitude>
	<RacistAttitude>
		<Race>RACE_TRIBAL</Race>
		<iAttitudeChange>-1</iAttitudeChange>
	</RacistAttitude>
</RacistAttitudes>

List of civics you want ai leaders to prioritize or to avoid using. Positive values make the ai favor certain civics while negative values make them avoid the civic.
<CivicAIWeights>
	<CivicAIWeight>
		<CivicType>CIVIC_PACIFISM</CivicType>
		<IAIWeight>10</IAIWeight> ///// A value like this serves as a gentle nudge for the leader to consider the civic more.
	</CivicAIWeight>
	<CivicAIWeight>
		<CivicType>CIVIC_SLAVERY</CivicType>
		<IAIWeight>-9000</IAIWeight> ///// A value this low effectively makes the ai never choose this civic. 
	</CivicAIWeight>
</CivicAIWeights>


New memory types:

MEMORY_DESTROYED_CIV ///// Triggered when a civ is destroyed, memory is towards the responible player(s) that have been met.
MEMORY_USED_NUKE ///// Triggered when nuking yourself or nuking a civ they are not friends with.
